## Context
用户希望 AI 能像真人伴侣一样更主动地聊天，并能“读懂”历史聊天记录截图中的内容。同时需要解决前端代码未同步到容器的问题。

## Goals
- 让 AI 在回复时根据语境主动延展话题。
- 将图片中的对话文字提取并整合进 `interests.md`。
- 确保“退出登录”和“切换恋人”按钮在界面显示。

## Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| 图像处理方式 | 文本化转录 | DeepSeek 为纯文本模型。通过将图片中的聊天记录转录为文本并附在 Markdown 中，可以最直接地让 AI “理解”这些背景。 |
| 主动性控制 | 提示词驱动 | 在 System Prompt 中加入指令：“不要只做答复机，要学会引导话题，像真实恋爱一样分享细节”。 |
| 部署方案 | 强制 Build | 前端为静态构建，必须重新执行 Docker Build 才能将新代码打入 Nginx 镜像。 |

## Risks / Trade-offs
- **[Risk]** AI 过度话痨 -> **Mitigation**: 在提示词中规定：“主动性应符合当前性格（如文静角色应表现为分享感受而非连续追问）”。
